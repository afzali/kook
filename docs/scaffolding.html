سبک سراسری برای سی اس اس reset، typography و links

نیاز به doctype اچ تی ام ال 5

کوک بر اساس اچ تی ام ال 5 و سی اس اس 3 ساخته شده است. برای همین در ابتدای تمامی پروژه های کوک از doctype استفاده بشود.

<!DOCTYPE html>
<html lang="fa">
  ...
</html>

تایپوگراف و لینک ها

در فایل scaffolding.less, ما نمایه عمومی، تایپوگراف و لینک ها را قرار دادیم. مهمترین آنها عبارتند از :

  • حذف مارجین صفحه
  • قرار دادن back-ground: white; در body
  • استفاده از خصوصیت @baseFontFamily و @baseFontSize و @baseLineHeight به عنوان پایه تایپوگرافی
  • قرار دادن رنگ سراسر لینک ها توسط @linkColor و قرار دادن زیر خط فقط در :hover

بازنشانی و حالت عادی

ما در کوک از کلاس های متفاوتی از جمله Normalize.css که توسط Nicolas Gallagher و همچنین از کلاس HTML5 Boilerplate استفاده کرده ایم و فایل جدیدی با نام reset.less ایجاد کرده ایم که بسیاری از المانهای غیرضروری از آن حذف شده و مجموعه مناسبی را ایجاد کرد ه ایم.

در سیستم گرید روان بجای استفاده از پیکسل از دزصد برای محاسبه ابعاد استفاده می شود. این سیستم نیز به مانند سیستم گرید ثابت به صورت های متفاوت واکنش نشان می دهد,چه در دستگاه های دیگر و یا ابعاد مانیتور های دیگر.

متغیرها در لس

همانطوری مشاهده می کنید در مطالب بالا کوک بری تمامی تنظیمات مهم از متغییرهایی در سی اس اس استفاده کرده است که شما می توانید تمامی آنها را در فایل variables.less پیدا و تنظیم نمایید.

چگونگی سفارشی سازی

سفارشی سازی یعنی تغییر در متغییرهای سه گانه @grid* و کامپایل مجدد کوک. برای تغییر شما می توانید موازد مورد نظرتان را در فایل variables.less تغییر داده و از یکی از راه های چهارگانه کامپایل استفاده نمایید. اگر شما تغییری در گرید دادید حتما آن را در فایل grid.less نیز اعمال کنید.

طراحی را واکنشی نگه دارید

سفارشی کردن گرید فقط در ابعاد 940 پیکسل کار خواهد کرد پس برای کار کردن به صورت واکنشی حتما فایل responsive.less را نیز ویرایش نمایید.

طراحی واکنشی استفاده از مدیا کوئری برای دستگاهها و ابعاد متفاوت

ما چه کرده ایم

سی اس اس مدیا کوئری این امکان را می دهد تا ما برای شرایط متفاوت سی اس اس های متفاوت را اجرا نماییم که معمولا بر روی min-width و max-width تمرکز دارد.

  • تغییر عرض ستون های گرید ما
  • جابجای حالت شناور و ابعاد آن
  • تغییر اندازه فونت و برخی الملنها با توجه به دستگاه

استفاده از این تکنیک برای آغاز کار برای کاربران موبایل مناسب است. شاید برای پروژه های بزرگ تر و پیچیده تر لازم باشد طراحی متفاوتی صورت گیرد.

دستگاههای پشتیبانی شده

کوک با تجمیع این ابزار در یک فایل، قابلیت نمایش متفاوت در دستگاه های زیر را می دهد:

نام عرض طرح بندی عرض ستون ها فاصله ستون ها
تلفن های هوشمند 480px and below ستون های روان، نه با عرض مشخص
تلفن هوشمند تا تبلت 767px and below ستون های روان، نه با عرض مشخص
تبلت های پرتریت 768px and above 42px 20px
پیش فرض 980px and up 60px 20px
صفحه نمایش بزرگ 1200px and up 70px 30px

متا تگ ضروری

برای اطمینان از صحت کار حتما این متا تگ را در صفحه خودتان قرار دهید.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

استفاده از مدیا کوئری

کوک به صورت اتوماتیک در فایل اصلی سی اس اس خودش، فایل طراحی واکنشی را قرار نداده است اما با چند روش بسیار ساده شما می توانید از آن استفاده نمایید:

  1. کامپایل فایل اختصاصی bootstrap-responsive.less
  2. افزودن @import به فایل اصلی سی اس اس کوک و کامپایل آن
  3. استفاده مستقیم از فایل bootstrap-responsive.cs آماده شده در پروژه

چرا خودمان قرار ندادیم?واقعیت این است که این سبک از طراحی در بسیاری از موارد جز اصلی نبوده و نیاز به استفاده از |آن نیست.

  // Landscape phones and down
  @media (max-width: 480px) { ... }

  // Landscape phone to portrait tablet
  @media (max-width: 768px) { ... }

  // Portrait tablet to landscape and desktop
  @media (min-width: 768px) and (max-width: 980px) { ... }

  // Large desktop
  @media (min-width: 1200px) { .. }

کلاس های سودمند طراحی واکنشی

این ها چه هستند

برای سرعت بخشیدن به طراحی های موبایل بیس، ابزارهایی هستند که به شما کمک می کند برخی از الملنها را پنهان یا پیدا نمایید.

کی استفاده نماییم

معمولا بدرد پروژه هایی می خورد که اختلاف اصلی در ساختار در دستگاههای متفاوت نداشته باشد و نه برای تغییرات بسیار کلی و پیچیده.

مثلا شما قصد نمایش امان <select> را در منوی موبایل دارید و نه در دسکتاب یا تبلت.

کلاس های پشتیبان کننده

کلاس هایی که برای استفاده در مدیا کوئری استفاده شده است را می توانید در اینجا ببینید (تفکیک شده توسط دستگاه). شما می توانید این ها را در فایل responsive.less بیابید.
